## 二、PID 控制器与一阶电机模型

PID 控制规律为：

```text
u(t) = Kp * e(t) + Ki * integral(e(t)) + Kd * de(t)/dt
```

其中：

| 参数 | 作用 |
|------|------|
| `Kp` | 比例项，根据当前误差快速产生控制输出 |
| `Ki` | 积分项，累积历史误差，用于消除稳态误差 |
| `Kd` | 微分项，根据变化趋势提供阻尼，抑制振荡 |

仿真电机采用一阶模型：

```text
G(s) = K / (tau * s + 1)
```

模型参数设置为 `K = 1.0`、`tau = 0.1 s`、采样周期 `dt = 1 ms`。初始目标位置为 1.0。

## 四、高级 PID 处理

### 1. 积分抗饱和

控制器输出被限制在 `[-limit, limit]`。当输出已经饱和时，不继续更新积分项，避免积分持续累积导致恢复阶段出现严重过冲。

```python
if output == raw_output:
    self.integral = candidate_integral
```

### 2. 微分滤波与微分突跳抑制

最初对误差 `error` 求导。目标值从 0 突然变为 1 时，误差变化很大，`high Kd` 曲线出现了不合理的起始突跳。

随后改为对测量值求导：

```python
raw_derivative = -(
    measurement - self.previous_measurement
) / self.dt
```

这样目标值阶跃不会直接产生巨大的微分输出。修改后，`high Kd` 曲线从 0 平稳起步，稳态误差也从 `0.162004` 降至 `0.147752`。

## 五、Ziegler-Nichols PID 整定

测试时设置：


### 2. 经典 Ziegler-Nichols 公式

PID 参数计算公式为：

```text
Kp = 0.6 * Ku
Ki = 2 * Kp / Tu
Kd = Kp * Tu / 8
```

|------|---:|
| Kp | 4.92 |
| Ki | 131.20 |
| Kd | 0.046 |

## 六、简化 FOC 仿真

FOC 的基本思想是将三相电流转换到随转子旋转的 `d-q` 坐标系：

```text
三相电流 (a, b, c)
    ↓ Clarke 变换
静止坐标系 (alpha, beta)
    ↓ Park 变换
旋转坐标系 (d, q)
```

其中：

| 轴 | 控制含义 |
|------|----------|
| d 轴 | 主要对应磁链控制，本次设定参考值为 0 |
| q 轴 | 主要对应转矩控制 |

本次实现了 Clarke 变换、Park 变换及其逆变换。外环速度 PI 控制器输出 `q` 轴电流参考值，简化电机模型根据 `q` 轴电流计算转矩：

```text
torque = 0.1 * iq
```

为了模拟实际 FOC 的优势，电流内环时间常数设置为 `5 ms`；直接 PID 的执行器转矩响应时间常数设置为 `40 ms`。在 `0.8 s` 时加入 2.0 的负载转矩阶跃。

需要注意：FOC 的速度 PI 输出单位为电流，直接 PID 输出单位为转矩。由于转矩常数为 0.1，FOC 的速度 PI 增益需要按单位关系放大 10 倍，才能与直接 PID 的转矩量级对应。

## 八、今日遇到的问题与解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `ModuleNotFoundError: No module named 'numpy'` | 虚拟环境未安装 NumPy 和 Matplotlib | 使用 `python -m pip install numpy matplotlib` 安装依赖 |

## 十、今日总结

今天完成了从基础 PID 参数分析到简化 FOC 对比的完整控制算法实验。

PID 参数实验表明，`Ki` 对消除稳态误差最关键；提高 `Kp` 可以提升响应速度，但不能替代积分项；`Kd` 则应配合测量值微分和滤波使用，避免目标值变化带来的微分突跳。

Ziegler-Nichols 实验得到 `Ku = 8.2`、`Tu = 0.075 s`，并计算出一组标准 PID 初始参数。但这组参数产生了约 74% 的超调，说明经典整定公式偏向快速响应，实际工程仍需要依据超调、负载扰动和执行器限制继续保守化调参。

在简化 FOC 仿真中，FOC 通过将控制量映射为 d-q 坐标系中的 q 轴电流，并假设快速电流内环，使转矩建立和负载补偿更及时。相比之下，直接 PID 结构更直观、实现简单，但对执行器响应速度更敏感。

这次实践让我认识到，控制算法不能只比较“是否到达目标值”，还需要同时观察上升速度、超调、稳态误差、负载扰动恢复能力、控制量大小和实际执行器约束。
